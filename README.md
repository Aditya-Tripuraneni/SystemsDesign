# Systems Design Learning Repository

This repository is my working space for learning and practicing systems design.
It brings together notes, low-level design implementations, and small experiments
that help me understand how common distributed systems ideas work in practice.

## Purpose

- Track progress while studying systems design concepts
- Implement low-level design ideas in code
- Collect useful references, patterns, and learning resources
- Keep small prototypes and exercises in one place

## What’s In Here

- `LLD/` - low-level design exercises and implementation work
- `RateLimitingAlgos/` - rate limiting experiments and related code
- Miscellaneous learning scripts and practice files at the repository root

## Topics I’m Exploring

- Rate limiting strategies
- Hashing and consistent hashing
- Design ideas and tradeoffs from *Designing Data-Intensive Applications* and *System Design Interview* by Alex Xu
- Scalable system components and tradeoffs

## Goal

The goal of this repo is not to be polished production software.
It is a study log: a place to build intuition, test ideas, and revisit concepts
as I learn more about system architecture and design.
